English version

SafePhone ("the app", "the service") is an Android mobile security application. This Privacy Policy explains what information the app handles, how it is stored, and what choices users have. SafePhone is designed to work locally on the user’s device and does not send personal data to external servers.

1. Information handled by the app

SafePhone does not collect, sell, share, or transmit personally identifiable information to external servers. Information required for the app’s functionality remains on the user’s device.

1.1 Information stored locally

  • Security PIN: The PIN used to arm/disarm the alarm is stored locally in the app’s private storage. It is not transmitted outside the device.
  • App settings: Selected alarm modes, sensor preferences, timer settings, and alarm sound preferences are stored locally.
  • Armed/disarmed state: The app may store whether the alarm is currently armed. This information is deleted when the app is uninstalled.

1.2 Permissions requested and why they are used

  • Camera: Used only to activate the device flashlight/strobe during an alarm. The app does not take, store, or transmit photos or videos.
  • Wake lock: Used to keep monitoring active while the screen is off, when the alarm is armed.
  • Vibration: Used for alarm vibration and warning feedback.
  • Foreground service: Required by Android to keep the anti-theft monitoring service active in the background while the alarm is armed.
  • Notifications: Used to display the persistent foreground service notification required by Android.
  • Display over other apps: Used to show the PIN unlock screen when the alarm is triggered.
  • Boot completed: May be used to restore monitoring after device restart if supported by the app configuration.
  • Network state / Wi‑Fi state: These permissions may be present due to the Android project configuration but SafePhone does not use them to collect or transmit user data.

1.3 Information not collected

SafePhone does not collect or share:

  • Name, email address, phone number, or postal address.
  • Location data.
  • Contacts, SMS, call logs, photos, files, or browsing history.
  • Advertising identifiers or analytics identifiers.
  • Account credentials or third-party login data.

2. Device sensors

SafePhone may access device sensors only to detect unauthorized movement or tampering while the alarm is armed. These sensors may include:

  • Accelerometer, for movement detection.
  • Proximity sensor, for pocket/bag detection.
  • Light sensor, for sudden exposure to light.
  • Headphone connection status.
  • Charging connection status.

Sensor readings are processed locally in real time and are not stored, transmitted, sold, or shared.

3. Third-party services

SafePhone does not use third-party analytics, crash reporting, advertising networks, cloud processing, or remote synchronization services. The app does not require an internet connection for its core functionality.

4. Data sharing

SafePhone does not share user data with third parties. The app does not sell user data and does not transfer personal data to external servers.

5. Data deletion

Users can delete all local app data by uninstalling SafePhone from the Android device. This removes the locally stored PIN and app preferences.

6. Children’s privacy

SafePhone is not specifically directed to children. The app does not knowingly collect personal information from children.

7. Security

Information used by SafePhone is stored in the app’s private local storage area on the user’s device. Users should change the default PIN, keep their Android device updated, and use Android’s built-in lock screen protection.

8. Changes to this policy

This Privacy Policy may be updated if the app’s data practices change. The latest version will be available at this URL, with the last updated date shown at the top of the page.

9. Contact

For questions about this Privacy Policy, contact the developer through the project’s public GitHub profile or repository issue tracker when available.

Versión en español

SafePhone ("la aplicación", "el servicio") es una aplicación de seguridad móvil para Android. Esta política describe qué información maneja la aplicación, cómo se almacena y qué derechos tiene el usuario. La aplicación está diseñada para funcionar de forma local en el dispositivo y no enviar datos personales a servidores externos.

1. Información que maneja la aplicación

SafePhone no recoge, vende, comparte ni transmite información personal identificable a servidores externos. La información necesaria para el funcionamiento de la app permanece en el dispositivo del usuario.

1.1 Información almacenada localmente

  • PIN de seguridad: El PIN usado para armar/desarmar la alarma se almacena localmente en el área privada de la aplicación. No se transmite fuera del dispositivo.
  • Preferencias de configuración: Modos de alarma seleccionados, sensores activos, temporizadores y preferencias de sonido se almacenan localmente.
  • Estado de armado: La app puede guardar si la alarma está armada o desactivada. Esta información se elimina al desinstalar la aplicación.

1.2 Permisos solicitados y uso

  • Cámara: Se usa únicamente para activar el flash/luz estroboscópica durante una alarma. La app no captura, almacena ni transmite fotos o vídeos.
  • Wake lock: Se usa para mantener la monitorización activa con la pantalla apagada cuando la alarma está armada.
  • Vibración: Se usa para vibración de alarma y avisos.
  • Servicio en primer plano: Requerido por Android para mantener activo el servicio de vigilancia mientras la alarma está armada.
  • Notificaciones: Se usan para mostrar la notificación persistente obligatoria del servicio en primer plano.
  • Mostrar sobre otras apps: Se usa para mostrar la pantalla de desbloqueo con PIN cuando salta la alarma.
  • Arranque completado: Puede usarse para restaurar la monitorización tras reiniciar el dispositivo si la configuración de la app lo permite.
  • Estado de red / Wi‑Fi: Estos permisos pueden estar presentes por la configuración del proyecto Android, pero SafePhone no los usa para recoger ni transmitir datos del usuario.

1.3 Información que NO se recoge

SafePhone no recoge ni comparte:

  • Nombre, email, teléfono o dirección postal.
  • Ubicación.
  • Contactos, SMS, llamadas, fotos, archivos o historial de navegación.
  • Identificadores publicitarios o de analítica.
  • Credenciales de cuentas externas.

2. Sensores del dispositivo

SafePhone puede acceder a sensores físicos únicamente para detectar movimiento o manipulación no autorizada mientras la alarma está armada. Estos sensores pueden incluir:

  • Acelerómetro, para detección de movimiento.
  • Sensor de proximidad, para detección de bolsillo/bolso.
  • Sensor de luz, para cambios bruscos de iluminación.
  • Estado de conexión de auriculares.
  • Estado de conexión de carga.

Las lecturas de sensores se procesan localmente en tiempo real y no se almacenan, transmiten, venden ni comparten.

3. Servicios de terceros

SafePhone no usa analítica de terceros, crash reporting remoto, redes publicitarias, procesamiento en la nube ni sincronización remota. La app no necesita conexión a internet para su funcionalidad principal.

4. Compartición de datos

SafePhone no comparte datos del usuario con terceros. La app no vende datos del usuario ni transfiere datos personales a servidores externos.

5. Eliminación de datos

El usuario puede eliminar todos los datos locales desinstalando SafePhone del dispositivo Android. Esto elimina el PIN y las preferencias locales de la aplicación.

6. Privacidad de menores

SafePhone no está dirigida específicamente a menores. La aplicación no recoge conscientemente información personal de menores.

7. Seguridad

La información usada por SafePhone se almacena en el área privada local de la aplicación dentro del dispositivo. Se recomienda cambiar el PIN por defecto, mantener Android actualizado y usar el bloqueo de pantalla nativo del sistema.

8. Cambios en esta política

Esta política puede actualizarse si cambian las prácticas de datos de la aplicación. La versión más reciente estará disponible en esta URL, con la fecha de última actualización indicada al inicio.

9. Contacto

Para preguntas sobre esta política de privacidad, contacte con el desarrollador a través del perfil público de GitHub o del sistema de issues del repositorio cuando esté disponible.
